# E2E Audit - 2026-03-30

## Scope

- Audited the Playwright suite in `e2e/`
- Brought the local backend up on `http://localhost:3001`
- Reproduced failures with the current repo state
- Fixed the E2E bootstrap and updated stale assertions to the current UI

## Initial Findings

1. `e2e/global-setup.ts` had two auth bootstrap defects:
   - `.env.e2e` always overwrote `process.env`, so a fresh CLI token could not override an expired local token
   - the setup only seeded `localStorage`, while the app boot path authenticates through `/api/auth/me` using the `tfi_auth_token` cookie
2. After fixing bootstrap, the remaining Playwright failures were mostly stale UI expectations rather than product regressions:
   - `Settings` tests assumed `Scheduler`, `System`, and `Audit` content was visible on the default `General` tab
   - `Live Monitor` tests still expected the old `Scheduler Control` / `Run Once` UI
   - dashboard/report/tracker KPI assertions still used older labels such as `Win Rate` and `ROI`

## Changes Applied

- Updated `e2e/global-setup.ts` so CLI-provided `E2E_TOKEN` wins over `.env.e2e`
- Updated `e2e/global-setup.ts` to seed both auth cookie and `localStorage`
- Realigned stale Playwright assertions with the current UI copy and tab routing
